- Added `NetworkStatusBanner` component for UI feedback

- Integrated offline fallback in WorkOrderService methods

**Key Features:**
- Queue persisted to UserDefaults (survives app restarts)
- Automatic retry when network connectivity returns
- Optimistic local updates for offline actions
- Maximum 5 retries before giving up on item
- Support for:
  - Photo uploads (saved to Documents/OfflinePhotos/)
  - Clock in/out actions
  - Task status updates
  - Work order status changes

**NetworkMonitor Features:**
- Real-time connectivity monitoring via NWPathMonitor
- Connection type detection (WiFi, Cellular, Ethernet)
- Notification posting for non-Combine consumers
- isGoodForUpload property for smart upload decisions

**UI Integration:**
- Offline indicator in MyWorkView navigation bar
- NetworkStatusBanner showing pending item count
- Syncing progress indicator
- Automatic queue processing on network restore

**Offline Flow:**
1. User performs action (clock in, complete task, add photo)
2. If offline, action queued locally with optimistic UI update
3. Banner shows "X pending" items
4. When network returns, queue auto-processes
5. Success haptic on sync completion
